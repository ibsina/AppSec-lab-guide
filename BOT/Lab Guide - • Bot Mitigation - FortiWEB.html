FortiABP

 

First, we need to explain what FortiABP is and what value it can bring. FortABP stands for FortiGuard Advanced Bot Protection.

FortiGuard Advanced Bot Protection is a dedicated bot mitigation technology with a multidimensional, deep-learning engine that learns and tracks bot attacks over time using sophisticated AI model training. Here are some of the critical ways that enable FortiGuard ABP to detect and stop malicious bots while allowing legitimate traffic through:

 

  • IP reputation database — Maintains a real-time database of known or suspicious IP addresses associated with bots and blocks traffic from them.
  • Browser Fingerprinting — Creates unique fingerprints for each visitor by cross-referencing various browser/device attributes to recognize repeat offenders. This includes detecting crawler-specific attributes and checking browser and OS inconsistencies.
  • Biometric detection — Analyzes visitors' device interactions to determine if a user is an actual human or a bot, as biometric signals are complicated for bots to fake. This includes monitoring client events (over 250 characters), mouse movements (such as scrolling behavior and clicks), and keyboard clicks.
  • Machine learning models — FortiGuard ABP uses AI to train models on vast datasets and continuously improve and refine bot detection capabilities.
  • Real-time threat intelligence (AI Score) — Leverages global threat intelligence to stay on top of new and emerging bot threats and update protections. The AI Score is compiled through deep learning and data correlation, multi-dimensional comparison, and multivariate data over time.
  • Comprehensive analytics — Provides detailed bot traffic analytics and attack forensics to enhance understanding of bot patterns and strategies.
  • Integration with FortiADC and FortiWeb — Allows FortiADC and FortiWeb to send telemetry data to the bot protection system, providing deeper insights into sophisticated bots for more accurate detection and blocking.

 

The factors listed above collectively contribute to calculating the Risk Score — a numerical value ranging from 0 to 100 that reflects the likelihood of a user being a bot.

 

Why FortiABP?

 

Why should customers look for this kind of solution? Perhaps the most familiar issue with bots is known for people who like to go to concerts, theater, sports events, or other events where limited seating is available. Bots run by 'criminals' buy tickets to quickly resell them on 2nd hand websites for higher prices. Blocking these bots will increase customer confidence in sales and prevent criminals from making money using these bots.

FortiADP Onboarding

FortiABP uses a process called PET (Professional Engagement Team) after you create an application on the FortiABP portal. The PET process (done by the FortiABP team) will investigate the application and build the necessary components to integrate the website with FortiABP. This process can take a couple of days. Please note that we re-use an existing application for this lab to speed up this process. If you run this lab with an application other than the existing applications in this workshop, please note that you must wait this PET process to finish.

 

See more here: PET process

3.2 FortiWeb Configuration

 

Create an Advanced Bot Protection Profile:

 

Go to Bot Mitigation > Advanced Bot Protection

Hint: If you are not seeing this, please do a hard refresh on your browser Windows(Shift+F5),

MacOS (command+r).

  • Create New
  • Name: emr

 

  • Application ID: copy openemr "Application ID" from ABP dashboard or paste the application ID you 'created' in the previous section (3.1)
  • Action: Alert & Deny
  • Severity: High
  • Bot Confirmation: Enable
  • Verification Method: CAPTCHA
  • Max Attempt Times: 3
  • Validation Timeout: 20 Seconds
  • OK

 

 

 

 

 

 

Apply the Advanced Bot Protection Profile to the Web Protection Profile.
Go to Policy > Web Protection Profile > Inline Protection Profile > Inline Standard Protection - EMR

  • Apply the emr Advanced Bot Protection Profile under the Bot Mitigation section
  • OK
